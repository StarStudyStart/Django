android_support_v7包

1、更新sdk中的Android support repository，否则可能不能使用一些最新的AppCompatActivity类。

               
          
2、导入v7包时要copy 工程的工作空间中，否则在其他项目引用的时候，可能老是添加不了。


          

3、v7项目的编译版本要在5.0以上，新建的项目也要在5.0以上

          
4、新导入的v7可能会出现一些错误。

I was having the same problem when I started adding Aviary into the project I´m working on.Finally I´ve managed to solve it by changing the style adding an extra line due to I didn´t wanted to change any line code from Aviary:
<style name="Theme.Base.AppCompat.Dialog.FixedSize" parent="Base.Theme.AppCompat.Dialog.FixedSize">        
    </style>
it seems that Base has been add so it changes completely the style name.
Before was Theme.Base.AppCompat.Dialog.FixedSize and now is Base.Theme.AppCompat.Dialog.FixedSize

btw, I´ve also changed my theme to:

 <style name="AppTheme" parent="Base.Theme.AppCompat.Light.DarkActionBar">
        <!-- Customize your theme here. -->
    </style>
 If you have this error due to the StyleTheme.Base.AppCompat.Dialog.FixedSize is not available. You can solve it by extend the styleBase.Theme.AppCompat.Dialog.FixedSize which is the same one but with different Parent.

5、 toolbar使用时出现错误：AppCompat does not support the current theme features。。。
          解决方法是：使用android预置的主题
升级到 AppCompat v22.1.0 报错: AppCompat does not support the current theme features

最新的Surpport里面的主题检查好像更严格了。以前使用

ActionBarActivity + Toolbar

的时候，我是这么定义的

<style name="AppTheme.Base" parent="Theme.AppCompat.Light">
        <item name="android:windowNoTitle">true</item>
        <item name="windowActionBar">false</item></style>

升级之后就出问题了。应该使用

<style name="AppTheme.Base" parent="Theme.AppCompat.Light">
        <item name="windowNoTitle">true</item>
        <item name="windowActionBar">false</item></style>

其实最好的方法还是，不管在哪里，都使用sdk预置的主题：

<style name="AppTheme.Base" parent="Theme.AppCompat.Light.NoActionBar"></style>

6、 要删除 新建项目的libs目录下v4还有v7jar包，否则应用程序可能会安装不了。

 7、在layout中调用时要使用类似这样“域名+控件”的方式    <android.support.v4.widget.DrawerLayout     />  来标明使用的是v4包或者v7包下的控件，以便兼容低版本。
